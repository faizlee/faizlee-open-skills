## 📝 维护指南

### 何时添加错题？

**必须添加**：
- 🔴 高严重程度问题（影响用户功能、信誉）
- 🟡 中等程度问题且反复出现（≥2次）

**建议添加**：
- 🟢 低严重程度但有教育意义的问题
- 流程改进相关的问题
- 有技术价值的问题

**无需添加**：
- 简单的拼写错误
- 一次性环境问题
- 显而易见的问题

### 错题格式要求

每道错题**必须**包含：
1. ✅ 元信息（编号、日期、严重程度）
2. ✅ 问题描述
3. ✅ 错误决策
4. ✅ 根本原因分析（至少3个原因）
5. ✅ 技术原因
6. ✅ 正确解决方案
7. ✅ 验证方法
8. ✅ 关键经验（至少3条）
9. ✅ 相关文档
10. ✅ 防范措施（至少3条）
11. ✅ 类似场景
12. ✅ 反思总结
13. ✅ 元信息（出现次数、状态）

### 编号规则

- 格式: `#[三位数字]`
- 示例: `#001`, `#002`, `#003`
- 规则: 按创建顺序递增

### 更新频率

- **添加**: 每次发现新问题时
- **审查**: 每月一次，更新状态
- **统计**: 每季度一次，分析趋势
- **归档**: 每年一次，归档已解决的旧错题

### 状态流转

```
待解决 → 已解决 → 持续监控 → 归档
   ↑                        ↓
   └────────────────────────┘
   （问题重现时）
```
